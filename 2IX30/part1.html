Five Dimensions of Responsible Data Science

Responsible Data Science is structured around five dimensions, sometimes remembered by the acronym FACT + Safety: Fairness, Accountability, Confidentiality, Transparency, and Safety.

Dimension Description
Safety Systems should not cause unintended harm; robustness against errors and adversarial inputs.
Fairness Systems should treat individuals and groups equitably; avoiding discrimination and harmful bias.
Accountability Developers and deployers can be held responsible for system behaviour and its consequences.
Confidentiality Personal and sensitive data is protected; privacy is preserved throughout the data lifecycle.
Transparency Systems and their decision processes are understandable and explainable to stakeholders.

The ML Pipeline

Dual-Track Pipeline
Real World:   Problem Formulation → Data Collection → Decision Making
Modeled World:   Training Data → Model → Predictions
Key Insight

Bias can enter at every transition point — from problem formulation to data collection to model training to decision making.

Six Types of Fairness-Related Harm

Fairness-related harms describe the ways in which a system can negatively impact individuals or groups. These six categories are not mutually exclusive — a single system can cause multiple types simultaneously.

Harm Type Definition Example
Allocation harm Unfair distribution of opportunities or resources (e.g., loan denial, hiring rejection). Loan denial for qualified applicants from a group.
Quality-of-service harm The system works less well for certain groups, even when access is nominally equal. Voice recognition failing for certain accents.
Representation harm Certain groups are underrepresented or misrepresented in system outputs or training data. Image search for “CEO” returning mostly white men.
Stereotyping harm The system reinforces or amplifies harmful social stereotypes about groups. Word embeddings associating “homemaker” with women.
Denigration harm Outputs are actively derogatory, offensive, or degrading toward specific groups. Tagging Black people as “gorillas” in image recognition.
Procedural harm Lack of transparency, recourse, or due process in automated decisions affecting individuals. No way to appeal an automated decision.

Four Types of Data Bias

Data bias arises when the training data does not faithfully or fairly represent the phenomenon a model is trying to capture. Each type has distinct causes and implications for downstream harm.

Bias Type Definition Example
Representation bias Training data does not reflect the true population; certain groups are over- or undersampled. Dataset with 90% male, 10% female.
Historical bias Past societal discrimination is encoded in the data, even if the data was collected correctly. Hiring data reflecting past gender discrimination.
Predictive bias Features have different predictive power across groups, leading to disparate model performance. Zip code predicts income well for one group but not another.
Aggregation bias A single model cannot capture the distinct data distributions of different groups, performing well on average but poorly for minorities. Medical model trained on pooled data fails for minorities.

Historical Bias Subtypes

Two mechanisms through which historical bias enters data

Life's Bias

The data accurately reflects a genuinely biased reality. The world itself is unfair — for example, certain groups historically received fewer opportunities — and the data faithfully records that unfairness. The measurement is correct, but the underlying construct is skewed.

Example: Past hiring records show fewer women in senior roles not because the data is wrong, but because women were systematically excluded.

Measurement Bias

The data is a biased measurement of the underlying construct. The world may not be perfectly fair, but the way we measure or proxy the outcome introduces additional distortion beyond what the true construct would show.

Example: Using arrest rates as a proxy for criminal behaviour introduces bias because policing is unevenly distributed across communities, so the proxy mismeasures the true underlying re-offense risk.

Bias Pipeline

The bias pipeline describes how bias accumulates as we move from the true latent potential of an individual to the data an ML model ultimately sees.

Pipeline Diagram
Innate Potential —[life's bias]→ Construct —[measurement bias]→ Measured Outcome

Innate Potential: the true, unbiased latent trait of an individual (e.g., actual job competence).
Construct: the real-world expression of that potential, shaped by societal structures and life's bias (e.g., career history).
Measured Outcome: the proxy label in the dataset, further distorted by measurement bias (e.g., salary, arrest record).

A model trained on Measured Outcome inherits both layers of bias. Removing the protected attribute alone does not eliminate either.

Bias-Harm Connections

Different types of data bias tend to produce different types of fairness-related harm. Understanding these pathways is essential for choosing appropriate mitigation strategies.

  • Representation bias + Historical bias → Allocation harm: groups that are underrepresented or historically disadvantaged receive fewer positive outcomes (jobs, loans, etc.).
  • Aggregation bias + Predictive bias → Quality-of-service harm: a single model optimises for the majority, performing disparately worse for minority groups.
  • Historical bias can cause Allocation harm even when no explicit protected attribute is used: if a statistical relationship exists between a sensitive attribute $A$ and the outcome $Y$ in the training data, the ML model will reproduce and sometimes amplify that relationship.
Key Takeaway

Bias is not a single monolithic problem. Diagnosing which type of bias is present determines which type of harm will result — and therefore which intervention (pre-processing, in-processing, or post-processing) is most appropriate. Treating all bias the same leads to ineffective or counterproductive fixes.

Key Case Studies

These real-world examples illustrate how different bias types manifest in practice and the harms they produce.

Case Study Bias Type(s) Harm(s)
Amazon Hiring Tool Historical bias (Life's bias) Allocation harm — the tool systematically downgraded résumés from women because historical hires were predominantly male.
COMPAS Recidivism Historical bias (Measurement bias) Allocation harm + Quality-of-service harm — arrest rates as proxy for re-offense risk introduced racially disparate false positive rates.
Google Image Search “CEO” Representation bias Representation harm — results overwhelmingly showed men, reinforcing a gendered stereotype about leadership.
Fraud Detection Cycle Selection → measurement → confirmation Self-reinforcing feedback loop — over-policed communities generate more fraud flags; those flags confirm the model’s prior, intensifying scrutiny in a vicious cycle.

Exercises

Exercise 1.1 — A facial recognition system misclassifies darker-skinned women at higher rates than white men. Which type of harm is this?

Answer: Quality-of-service harm — the system performs worse for certain groups.

Exercise 1.2 — A resume screening tool selects resumes of male applicants more often. Which type of harm is this?

Answer: Allocation harm — disproportionate allocation of opportunities (job interviews).

Exercise 1.3 — A healthcare algorithm uses insurance costs as a proxy for health needs. Black patients historically had less access to healthcare, so they incur lower costs despite equal or greater needs. What type of bias is this?

Answer: Historical bias, specifically measurement bias — costs are a biased measurement of the true construct (health needs).

Exercise 1.4 — You are a data scientist developing a financial risk model for small business loans. Name 2 potential benefits and 2 risks, each with the affected stakeholder.

Benefits:

  • Better/faster decision-making [bank, borrowers]
  • Explainable model allows borrowers to seek redress [borrowers]

Risks:

  • Inaccurate predictions lead to suboptimal decisions [bank, borrowers]
  • Model disproportionately rejects loans for certain groups [borrowers]
Exercise 1.5 — Explain how aggregation bias can result in quality-of-service harm.

Answer: Aggregation bias occurs when a single ML model cannot capture the distinct data distributions of different sensitive groups. It fits the majority well but not minorities. This leads to worse predictive performance for minority groups = quality-of-service harm.
