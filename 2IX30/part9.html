Definition

Definition — Federated Learning

Distributed ML where the model is trained across multiple clients without sharing raw data. Data stays at the source.

FedAvg Algorithm

  1. Sample subset $S$ of $M$ clients
  2. Send global model $w^t$ to selected clients
  3. Local updates: $K$ steps of SGD — $w_i^{k+1} = w_i^k - \eta \nabla L_i(w_i^k)$
  4. Aggregate:
Aggregation Formula
$$w^{t+1} = \sum_{i \in S} \frac{n_i}{N_S} w_i^K$$

where $n_i$ is the number of data points on client $i$ and $N_S = \sum_{i \in S} n_i$ is the total data points across sampled clients.

Global Loss

The global objective is a weighted average of local loss functions, weighted by each client's share of the total data.

$$L(w) = \sum_{m=1}^{M} \frac{n_m}{N} L_m(w)$$

where $N = \sum_{m=1}^{M} n_m$ is the total number of data points across all $M$ clients.

Challenges

Challenge Description Consequence
Statistical heterogeneity Non-IID data across clients — each client's distribution differs from the global distribution. Model may not converge to a good global optimum.
Client drift More local steps $K$ cause each client's model to move toward its own local optimum. Models drift away from the global optimum; aggregation degrades.
System heterogeneity Clients differ in compute power, memory, and connectivity. Some clients are stragglers. Slow clients delay training rounds or must be dropped, introducing bias.
Communication cost More rounds of communication improve convergence but are expensive over bandwidth-limited links. Trade-off between accuracy and total communication overhead.
Key Insight

Key trade-off: More local updates $K$ = less communication but worse convergence (more drift).

DP in FL

Differential Privacy can be applied to federated learning to bound what the server learns from each client's update.

  1. Clip updates: enforce $\|w_i^K - w^t\|_2 \leq C$ (L2 norm bounded by $C$)
  2. Sensitivity $\leq C / N_S$
  3. Add Gaussian noise to the aggregate
$$w^{t+1} = \sum_{i \in S} \frac{n_i}{N_S} \text{clip}(w_i^K) + \mathcal{N}(0, \sigma^2 I)$$

Privacy levels:

Example-level DP

Protects individual data points within a client's dataset. Assumes the server knows which client contributed and only hides per-sample information. Less noise required.

User-level DP

Protects an entire client's contribution — the server cannot determine whether a given client participated at all. Higher sensitivity means more noise is needed. Stronger but more costly guarantee.

Secure Aggregation

Definition — Secure Aggregation

A cryptographic protocol that allows the server to compute the aggregate of client updates without seeing any individual client's update.

  • Protects against an honest-but-curious server — one that follows the protocol but tries to infer private data from what it receives.
  • Does NOT protect against a malicious server that deviates from the protocol.
  • Complementary to Differential Privacy: secure aggregation hides individual updates; DP bounds inference from the aggregate.

Three-Way Trade-off

Federated learning involves a fundamental tension between three desiderata. Improving any one of them typically degrades at least one of the others.

Trade-off Triangle

Performance ↔ Communication Cost ↔ Privacy

  • More local steps reduces communication cost but increases client drift (hurts performance).
  • Stronger DP (more noise) preserves privacy but degrades model performance.
  • More communication rounds improves performance but raises communication cost and privacy exposure.
