XAI Taxonomy

Explainability methods can be classified along three orthogonal axes: how deeply they access the model internals, what scope of the model they explain, and whether they are tied to a specific model type.

Axis Category A Category B
Translucency Intrinsic — the model is interpretable by design (e.g., linear regression, decision tree). Post-hoc — a separate explanation is derived after training (e.g., SHAP, LIME).
Scope Global — explains the overall model behaviour across all inputs. Local — explains the model’s prediction for a single instance.
Portability Model-specific — exploits internal structure of one model family (e.g., tree-based Gini importance). Model-agnostic — treats the model as a black box; applicable to any model (e.g., PDP, ICE).

Motivations for XAI

Explanations serve different stakeholders and purposes. The following six motivations capture the main reasons explainability is required in practice.

  1. Model validation — verify that the model has learned the right patterns, not spurious correlations.
  2. User acceptance — affected individuals are more willing to trust and act on a decision they can understand.
  3. Fairness assessment — check whether sensitive attributes or proxies drive predictions inappropriately.
  4. Knowledge discovery — extract novel, actionable insights from the data that experts may not have anticipated.
  5. Justification — satisfy regulatory or legal requirements (e.g., EU AI Act, GDPR right to explanation).
  6. Trust calibration — help users understand when to rely on the model and when to override it.

Linear Regression

Linear regression is the canonical intrinsically interpretable model for continuous targets. Its coefficients directly encode feature effects.

$$y = \beta_0 + \beta_1 x_1 + \dots + \beta_n x_n$$
Interpretation

$\beta_j$ is the expected change in $y$ when $x_j$ increases by 1 unit, holding all other features constant (ceteris paribus).

On normalized data (zero mean, unit variance), 1 unit corresponds to 1 standard deviation. Coefficients are then on a common scale and directly comparable across features.

Pros

  • Coefficients have a clear, direct causal-style interpretation.
  • Confidence intervals and p-values are readily available.
  • Globally consistent — the same explanation applies everywhere.
  • Very fast to train and apply even at large scale.

Cons

  • Assumes a linear relationship; cannot capture nonlinear effects without manual feature engineering.
  • Cannot model feature interactions without explicit interaction terms.
  • Correlated features produce unstable, unreliable coefficient estimates.
  • Extrapolates beyond training data range, which can be dangerous.

Logistic Regression

Logistic regression extends linear regression to binary classification by modelling the log odds of the positive class.

$$\log\left(\frac{p}{1-p}\right) = \beta_0 + \beta_1 x_1 + \dots + \beta_n x_n$$
Interpretation

Coefficients are log odds ratios. Increasing $x_j$ by 1 unit changes the log odds by $\beta_j$, or equivalently multiplies the odds by $e^{\beta_j}$.

Example: if $\beta_j = 0.4$, then $e^{0.4} \approx 1.49$ — the odds of the positive outcome are 49 % higher for each additional unit of $x_j$.

Warning

Normalization completely changes apparent feature importance rankings! Comparing raw coefficients across features with different scales is meaningless. Always normalize before comparing magnitudes.

Decision Trees

A decision tree partitions the feature space recursively by greedily choosing the split that maximizes the reduction in impurity (e.g., Gini or entropy) at each node.

How Trees Are Built

Greedy, top-down construction: at each node, evaluate every possible split on every feature and choose the one that maximizes purity gain. The process repeats recursively on each child node until a stopping criterion is met (max depth, min samples, or zero impurity).

Pros

  • Easily visualized as a flowchart that domain experts can read directly.
  • Naturally captures nonlinear decision boundaries without feature engineering.
  • Captures feature interactions: the effect of a feature can differ across branches.
  • Handles mixed data types (continuous and categorical) with minimal preprocessing.

Cons

  • Unstable: small changes in training data can produce a completely different tree structure.
  • No extrapolation: predictions are constant within leaves; the tree cannot predict beyond the training data range.
  • Prone to overfitting: deep trees memorize noise; pruning or ensemble methods are typically required.
  • Greedy splitting is locally optimal but not globally optimal.

Global Surrogate

A global surrogate is an interpretable model (e.g., linear regression or shallow decision tree) that is trained to approximate the predictions of a black-box model, not the true labels.

Fidelity

Fidelity measures how well the surrogate replicates the black-box predictions (e.g., $R^2$ between surrogate output and black-box output). High fidelity means the surrogate closely mimics the black box.

However, high fidelity does not mean faithful representation of the black box’s internal reasoning. A surrogate with high $R^2$ may still fail to capture the true mechanisms the black box uses, especially in regions of the input space that are underrepresented in the training data.

Key Insight

The surrogate explains the black-box’s predictions, not the black-box itself. If the black box is wrong, the surrogate inherits that wrongness. Surrogates are useful for communicating behaviour, not for auditing internal correctness.

Gini Importance

Gini importance (also called Mean Decrease in Impurity, MDI) is a model-specific global feature importance measure for tree-based models.

Definition

For each feature, sum the total Gini impurity reduction across all nodes where that feature is used as a split point, weighted by the fraction of samples reaching each node. The result is then normalized so all importances sum to 1.

Formally: $I(f) = \sum_{t: \text{split on } f} \frac{n_t}{N} \cdot \Delta\text{Gini}(t)$, normalized over all features.

Bias Warning

Gini importance is biased toward continuous and high-cardinality features. Features with many possible split points have more opportunities to reduce impurity at each node, inflating their apparent importance even if they are not truly more informative.

PDP — Partial Dependence Plot

A Partial Dependence Plot (PDP) shows the marginal average effect of one (or two) features on the model’s predicted outcome, averaging out the influence of all other features.

$$\hat{f}_j(x_j) = \frac{1}{n}\sum_{i=1}^{n} f(x_j,\, x_{-j}^{(i)})$$
How It Works

For a fixed value $x_j$, replace the $j$-th feature for every training instance with that value, run the model on all $n$ modified instances, and average the predictions. Repeating this for a grid of $x_j$ values produces the PDP curve.

Limitation: OOD and Interactions

PDPs average over the marginal distribution of other features, which can create out-of-distribution (OOD) data points when features are correlated — combinations that never appear in reality. More fundamentally, PDPs do not capture feature interactions: if the effect of $x_j$ varies across instances (interaction with another feature), the PDP shows only the average and conceals that variation.

ICE — Individual Conditional Expectation

An ICE plot disaggregates the PDP by drawing one curve per training instance, showing how each individual instance’s prediction changes as $x_j$ varies.

ICE vs. c-ICE

A standard ICE plot can be hard to read when curves are at different vertical positions. Centered ICE (c-ICE) subtracts each curve’s value at a reference point (usually the minimum of $x_j$), anchoring all curves at zero at that point and making it easier to compare slopes and shapes.

Detecting Interactions

If ICE lines are parallel, there is no interaction between $x_j$ and other features: the effect of $x_j$ is the same for all instances. If ICE lines are non-parallel (cross or diverge), there is an interaction: the effect of $x_j$ depends on the value of some other feature.

Key Insight

ICE reveals interactions that PDP hides — but doesn’t tell you WHICH feature is involved. To identify the interacting feature, you need to colour ICE lines by candidate features and look for systematic patterns.
