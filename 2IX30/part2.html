Confusion Matrix

The confusion matrix summarises the four possible outcomes of a binary classifier with predicted label $\hat{Y}$ and true label $Y$.

Predicted $\hat{Y}=1$ Predicted $\hat{Y}=0$
Actual $Y=1$ TP — True Positive FN — False Negative
Actual $Y=0$ FP — False Positive TN — True Negative

Performance Metrics

All group fairness criteria are defined in terms of differences in these metrics between groups.

Metric Formula Intuition
Accuracy $\dfrac{TP + TN}{TP + FP + TN + FN}$ Fraction of correct predictions
FPR (False Positive Rate) $\dfrac{FP}{FP + TN}$ Rate of false alarms among negatives
FNR (False Negative Rate) $\dfrac{FN}{TP + FN}$ Rate of missed positives among positives
TPR / Recall $\dfrac{TP}{TP + FN}$ Fraction of true positives correctly identified; $\text{TPR} = 1 - \text{FNR}$
TNR / Specificity $\dfrac{TN}{TN + FP}$ Fraction of true negatives correctly identified; $\text{TNR} = 1 - \text{FPR}$
PPV / Precision $\dfrac{TP}{TP + FP}$ Fraction of predicted positives that are truly positive
NPV (Negative Predictive Value) $\dfrac{TN}{TN + FN}$ Fraction of predicted negatives that are truly negative
Selection Rate $\dfrac{TP + FP}{TP + FP + TN + FN}$ Fraction predicted positive (regardless of true label)
Base Rate $\dfrac{TP + FN}{TP + FP + TN + FN}$ Fraction that are truly positive in the population

Three Fairness Criteria

Let $A$ be the sensitive attribute (e.g., race, gender), $Y$ the true label, and $\hat{Y}$ the predicted label.

1. Demographic Parity (Independence)

$$P(\hat{Y} = y \mid A = a) = P(\hat{Y} = y \mid A = a')$$

  • Selection rates must be equal across groups.
  • Does not use the true label $Y$ — purely a property of the predictions.
  • Violation leads to allocation harm risk: one group receives a beneficial outcome less often.
  • Appropriate when you believe groups are equally qualified or that historical disparities reflect measurement bias or unjust historical context.
2. Equalized Odds (Separation)

$$P(\hat{Y} = y \mid A = a, Y = y) = P(\hat{Y} = y \mid A = a', Y = y) \quad \forall y$$

  • Both FPR and FNR (equivalently, TPR) must be equal across groups.
  • Uses the true label $Y$ — conditions on the ground truth.
  • Violation leads to quality-of-service harm: one group receives worse predictions for the same true outcome.
  • Appropriate when base rate differences across groups are considered legitimate.

Equal Opportunity (relaxation)

Requires only equal TPR — i.e., equal benefit for those who truly deserve a positive prediction: $$P(\hat{Y} = 1 \mid A = a, Y = 1) = P(\hat{Y} = 1 \mid A = a', Y = 1)$$ This is weaker than equalized odds because it ignores FPR equality.

3. Equal Calibration (Sufficiency)

$$P(Y = y \mid A = a, \hat{Y} = y) = P(Y = y \mid A = a', \hat{Y} = y)$$

  • Equal PPV and NPV across groups.
  • A prediction score of, say, 0.7 should mean the same probability of being positive for all groups.
  • Commonly used in recidivism and credit scoring contexts.
Conditional Demographic Parity

A refinement of demographic parity that conditions on a set of legitimate features $L$ (e.g., qualifications): $$P(\hat{Y} = y \mid A = a, L = \ell) = P(\hat{Y} = y \mid A = a', L = \ell)$$ This allows selection rate differences that are entirely explained by legitimate features.

Key insight — what each criterion conditions on

The three criteria can be understood through their conditional independence structure:

Criterion Independence statement Conditioned on
Demographic Parity $A \perp \hat{Y}$ Nothing — prediction is independent of $A$
Equalized Odds $A \perp \hat{Y} \mid Y$ True label $Y$
Equal Calibration $A \perp Y \mid \hat{Y}$ Predicted label $\hat{Y}$

Graphically: $A \to Y \to \hat{Y}$. DP blocks the $A \to \hat{Y}$ path; EO conditions on $Y$; EC conditions on $\hat{Y}$.

Incompatibility Theorem

Incompatibility Theorem

When base rates differ across groups and the classifier is imperfect, Demographic Parity + Equalized Odds + Equal Calibration cannot all hold simultaneously.

(Chouldechova 2017; Kleinberg et al. 2016.) At most two of the three criteria can be satisfied at once, except in degenerate cases where base rates are equal or the classifier is perfect.

Pairwise incompatibilities & proof sketch

Which pairs conflict?

Pair Compatible? Condition for compatibility
DP + Equal Calibration Only if $A \perp Y$ Base rates are equal across groups
DP + Equalized Odds Only if $A \perp Y$ Base rates are equal across groups
EO + Equal Calibration Only if perfect classifier No errors: FPR = FNR = 0 for all groups

Proof sketch (EO vs. Equal Calibration)

Assume groups $a$ and $a'$ have different base rates: $P(Y=1 \mid A=a) \neq P(Y=1 \mid A=a')$.

Equal Calibration requires $P(Y=1 \mid \hat{Y}=1, A=a) = P(Y=1 \mid \hat{Y}=1, A=a')$, i.e., the same PPV. Applying Bayes' theorem: $$\text{PPV}_a = \frac{\text{TPR}_a \cdot p_a}{\text{TPR}_a \cdot p_a + \text{FPR}_a \cdot (1-p_a)}$$ where $p_a = P(Y=1 \mid A=a)$ is the base rate of group $a$. If $p_a \neq p_{a'}$ and both $\text{FPR}$ and $\text{TPR}$ are the same (equalized odds), then $\text{PPV}_a \neq \text{PPV}_{a'}$ unless the classifier is perfect ($\text{FPR}=0$). Hence EO and EC are mutually exclusive when base rates differ and the classifier is imperfect.

Proof sketch (DP + EC)

  1. DP requires $A \perp \hat{Y}$
  2. EC requires $A \perp Y \mid \hat{Y}$
  3. By the contraction rule of conditional independence, these together imply $A \perp Y$
  4. When base rates differ: $P(Y=1 \mid A=a) \neq P(Y=1 \mid A=a')$, so $A \not\perp Y$
  5. Contradiction ⇒ DP + EC cannot both hold
Normative consequence

You MUST choose which metric to prioritize — this is a normative decision, not a technical one. No algorithm can simultaneously satisfy all three criteria when groups have unequal base rates. The choice reflects what kind of fairness you believe matters most in the specific application context.

When to Use Which Metric

Metric Use when… Underlying normative stance
Demographic Parity You believe measurement bias inflated observed differences, or that disparities stem from unjust historical/life-context bias and should be corrected. "Everybody is (or should be treated as) equal." Differences in outcomes must be erased regardless of observed predictors.
Equalized Odds Base rate differences between groups are considered legitimate (reflecting genuine differences in the quantity being predicted), and you want to give individuals with the same true label the same chance of being correctly classified. "Same true outcome deserves the same prediction." Groups are allowed to receive different overall selection rates if their base rates differ.
Equal Calibration Predictions are used as probability estimates (e.g., risk scores) and you need scores to be equally reliable across groups — a score of $p$ should mean the same probability of the outcome for everyone. "Predictions should carry the same meaning for all groups." Commonly required in high-stakes scoring (credit, recidivism).

Challenges

  • Defining the sensitive attribute: Who counts as a member of group $a$ vs. $a'$? Attributes like race or gender are socially constructed and self-reported, introducing measurement issues.
  • Multiple sensitive attributes: Intersectionality — satisfying a fairness criterion for gender and race separately does not guarantee it holds for Black women as a subgroup.
  • Label quality: If the true label $Y$ is itself biased (e.g., arrest records reflecting policing bias rather than actual crime), criteria that condition on $Y$ (EO, EC) can entrench bias.
  • Group vs. individual fairness: Meeting group-level parity says nothing about the fairness of individual decisions. Two individuals with identical features may still receive different predictions.
  • Gaming and proxy attributes: Removing $A$ from the feature set does not prevent discrimination if other features (zip code, surname) are correlated with $A$.
  • Dynamic effects: A classifier satisfying fairness at deployment time may create feedback loops that erode fairness over time (e.g., more surveillance leads to more arrests in the same group).
  • Operationalisation costs: Achieving fairness sometimes reduces overall accuracy; the trade-off must be evaluated in context.
  • Privacy constraints: Access to sensitive features is limited by privacy regulations (GDPR).
  • Small sample sizes: Small sample sizes for minority groups lead to imprecise estimates (wide confidence intervals).
  • Within-group heterogeneity: Group metrics may conceal within-group differences.
  • Construct validity: How sensitive features are measured matters (self-reported vs. observed).

Exercise 2.2 — Is it likely that representation bias is present if a dataset has: male=5, female=5, non-binary=4?

Answer: Compared to the general population, non-binary people are vastly overrepresented (~29% vs <1%). This points to potential representation bias.

Exercise 2.3 — A dataset shows base rates: male = 0.8, female = 0.6, non-binary = 0.25. Is historical bias likely?

Answer: Yes. Non-binary individuals have a substantially lower base rate, suggesting historical bias against that group.

Exercise 2.4 — A classifier has accuracy: male=0.8, female=0.8, non-binary=0.5. Is predictive bias likely?

Answer: Yes. Performance is much lower for non-binary individuals, pointing to predictive bias.

Exercise 2.5 — Explain how historical bias can result in allocation harm.
  1. Historical bias means a relationship exists between $A$ and $Y$ in the data.
  2. The ML model reproduces this pattern.
  3. If $\hat{Y}$ drives allocation decisions, reproducing historical bias causes unequal allocation = allocation harm.
Exercise 2.6 — When base rates differ, an imperfect classifier cannot simultaneously satisfy: (a) DP and conditional DP  (b) EO and DP  (c) EO and EC  (d) EC and EO

Answer: (c) Equalized odds and equal calibration. This is the incompatibility theorem.

The base rates differ: males have base rate 4/5 = 0.80, females have base rate 2/5 = 0.40.
