No Fairness Through Unawareness

A naive mitigation strategy is to simply remove the sensitive attribute $A$ (e.g., race, gender) from the feature set before training. This is called fairness through unawareness. Unfortunately, it does not work.

Warning
Simply removing $A$ is INSUFFICIENT. Proxy variables carry group information (zip code → race, name → gender). A model trained without $A$ can still discriminate indirectly through these correlated features.

Proxy variables are features that are not $A$ themselves but are statistically correlated with $A$. Because of this correlation, a model can reconstruct group membership from the remaining features and produce biased predictions. True fairness interventions must actively account for — not merely ignore — the sensitive attribute.

Pre-Processing

Pre-processing methods modify the training data before any model is fitted. They are model-agnostic: once the data is transformed, any classifier can be used downstream.

Relabeling

Relabeling addresses disparate selection rates by flipping a carefully chosen set of labels in the training data. The goal is to equalize $P(\hat{Y}=1 \mid A=a)$ across groups without destroying too much information.

Formula — Number of labels to flip

Let $D_{A=a}$ be the deprived group and $D_{A=a'}$ the favored group. The number of labels to flip is:

$$M = \frac{[P(Y=1|A=a') - P(Y=1|A=a)] \cdot |D_{A=a}| \cdot |D_{A=a'}|}{N}$$

where $N$ is the total dataset size.

The flip strategy works in two directions:

  • Promote (deprived group): select $M$ instances from the deprived group with $Y=0$ and the highest classifier confidence scores. Flip their labels to $Y=1$. These are borderline negatives that the model nearly classified as positive.
  • Demote (favored group): select $M$ instances from the favored group with $Y=1$ and the lowest classifier confidence scores. Flip their labels to $Y=0$. These are the least deserving positives in the favored group.

After relabeling, retrain the classifier on the modified dataset. The method is typically iterated until demographic parity is achieved.

Reweighing

Instead of changing labels, reweighing assigns a weight $W(x)$ to each training instance. Instances from underrepresented (deprived) subgroups receive higher weights; instances from overrepresented (favored) subgroups receive lower weights. The labels remain unchanged.

Formula — Instance weight

For an instance $x$ with sensitive attribute value $A=a$ and label $Y=y$:

$$W(x) = \frac{P(A=a) \cdot P(Y=y)}{P(A=a \wedge Y=y)}$$

This is the ratio of expected frequency (independence assumption) to observed frequency. $W = P_{\text{exp}} / P_{\text{obs}}$.

If a subgroup $(A=a, Y=y)$ is overrepresented in the data ($P_{\text{obs}} > P_{\text{exp}}$), then $W < 1$ — its influence on the model is reduced. If a subgroup is underrepresented ($P_{\text{obs}} < P_{\text{exp}}$), then $W > 1$ — its influence is amplified. The result is that the weighted empirical distribution satisfies demographic parity by construction.

Representation Learning

Representation learning seeks a mapping $X \mapsto Z$ such that the new feature space $Z$ is simultaneously fair and useful. Three competing objectives must be balanced:

  1. Group information is lost: $Z$ should not allow accurate reconstruction of $A$. Formally, $A \perp Z$ (or minimized mutual information $I(A; Z)$).
  2. Original information is preserved: $Z$ should allow reconstruction of $X$, so that useful features are retained (low reconstruction error $\|X - \hat{X}\|$).
  3. Prediction quality is maintained: $Z$ should allow accurate prediction of the target $Y$ (low classification loss $\mathcal{L}(Y, f(Z))$).
Prototype assignment

A common approach learns a set of $k$ prototypes $\{v_1, \ldots, v_k\}$ and represents each instance by its soft assignment vector. For instance $x_i$, the assignment to prototype $j$ is:

$$\alpha_{ij} = \frac{\exp(-d(x_i, v_j))}{\sum_{l=1}^{k} \exp(-d(x_i, v_l))}$$

where $d(\cdot, \cdot)$ is a distance function. The representation $z_i = \sum_j \alpha_{ij} v_j$ is the prototype-weighted embedding.

In-Processing

In-processing methods modify the learning algorithm itself to incorporate fairness as an objective during training. They require access to the training procedure.

Regularization

Fairness can be incorporated as an additional regularization term in the loss function. The objective to maximize is:

Formula — Fairness-regularized objective
$$\mathcal{L} = -L(D,\Theta) + \frac{\lambda}{2}\|\Theta\|^2 + \eta \cdot R(D,\Theta)$$
Term Role
$-L(D,\Theta)$ Negative log-likelihood (maximizing fit to data). Negated because we maximize this objective.
$\frac{\lambda}{2}\|\Theta\|^2$ L2 regularization on model parameters. Controls model complexity and prevents overfitting.
$\eta \cdot R(D,\Theta)$ Fairness regularizer. $R$ measures the degree of unfairness (e.g., disparate impact, disparate misclassification). $\eta$ controls the fairness-performance trade-off.
Key Insight
$\lambda$ is the L2 regularization parameter — it prevents overfitting by penalizing large weights. It does NOT control fairness. The fairness-performance trade-off is governed entirely by $\eta$: a larger $\eta$ enforces stronger fairness at the cost of predictive accuracy.

Adversarial Learning

Adversarial fairness uses a two-player game between a predictor and an adversary:

  • Predictor: learns to predict $\hat{Y}$ from $X$ as accurately as possible, while simultaneously preventing the adversary from recovering $A$.
  • Adversary: tries to predict the sensitive attribute $A$ from the predictor's output. Its success is a signal of residual discrimination.

The inputs to the adversary differ depending on the fairness criterion being enforced:

Fairness Criterion Adversary's Input Intuition
Demographic Parity (DP) $\hat{Y}$ only The adversary must not be able to infer $A$ from predictions alone. Predictions must be independent of $A$.
Equalized Odds (EO) $\hat{Y}$ and $Y$ The adversary must not be able to infer $A$ from predictions even when the true label is known. Error rates must be equalized across groups.

At convergence (Nash equilibrium), the predictor's output carries no information about $A$ beyond what is already captured by the criterion being enforced. The predictor and adversary are trained jointly with gradient reversal: gradients from the adversary loss are negated when updating the predictor.

Decoupled Classification

Decoupled classification trains a separate classifier for each group defined by the sensitive attribute $A$. The classifier for group $a$ is fitted only on data with $A=a$, allowing different decision boundaries per group.

Advantage: each sub-classifier can be optimally tuned for its group's data distribution — group-specific feature importance, thresholds, and model families are all possible.

Important Limitation
Decoupled classification does NOT ensure that predictions are independent of $A$. If $A$ correlates with $Y$ in the data, the group-specific classifiers will reproduce this pattern. Using $A$ at prediction time also raises legal and ethical concerns in protected-attribute settings.

Post-Processing

Post-processing methods adjust the predictions of an already-trained model. They require access to the sensitive attribute $A$ at prediction time but are model-agnostic — any black-box classifier can be wrapped.

Group-Specific Thresholds

A standard classifier applies a single threshold $\tau$ to convert scores into binary predictions: $\hat{Y} = \mathbf{1}[f(x) \geq \tau]$. Group-specific thresholding uses a different threshold $\tau_a$ per group: $$\hat{Y} = \mathbf{1}[f(x) \geq \tau_a]$$ By choosing $\tau_a$ appropriately for each group $a$, we can equalize a chosen fairness criterion (e.g., equalized odds) across groups.

When the required operating point does not correspond to a single observed threshold, randomized thresholds are used. A randomized classifier mixes two adjacent points on the group's ROC curve via linear interpolation: with probability $p$ use threshold $\tau_1$ and with probability $1-p$ use threshold $\tau_2$. This allows any point on the convex hull of the ROC curve to be achieved exactly.

Note
Finding the optimal group-specific thresholds is a linear program: minimize overall error subject to the chosen fairness constraint (e.g., equal TPR and FPR across groups). The solution may require randomization.

Reject Option Classification

Reject option classification identifies a critical region around the decision boundary — instances where the model is uncertain — and applies a fairness-aware rule within that region.

Critical Region

An instance $x$ with classifier score $f(x) \in [0,1]$ lies in the critical region if:

$$|f(x) - 0.5| \leq \theta_c$$

where $\theta_c \in [0, 0.5]$ is the critical region half-width. Instances outside this region keep their original predicted label.

Within the critical region, predictions are reassigned based on group membership:

  • Instances from the deprived group are assigned $\hat{Y} = 1$ (favorable outcome).
  • Instances from the favored group are assigned $\hat{Y} = 0$ (unfavorable outcome).
Definition: Favored Group
The favored group is the group with the higher selection rate according to the original (uncorrected) classifier — i.e., $\arg\max_a P(\hat{Y}=1 \mid A=a)$. The deprived group has the lower selection rate.

Leaf Relabeling

Leaf relabeling is a post-processing technique specific to decision tree classifiers. After training, the prediction at each leaf can be flipped independently without changing the tree structure. The algorithm greedily selects which leaves to relabel:

  1. For each leaf, compute the change in demographic parity disparity ($\Delta\text{dpd}$) and the change in accuracy ($\Delta\text{acc}$) if that leaf's label were flipped.
  2. Score each leaf by the ratio $\Delta\text{dpd} / \Delta\text{acc}$ — the fairness gain per unit accuracy loss.
  3. Greedily flip the leaf with the highest score. Repeat until the desired fairness level is reached or accuracy degrades unacceptably.
Greedy Selection Criterion
$$l^* = \arg\max_{l} \frac{\Delta\text{dpd}_l}{\Delta\text{acc}_l}$$

(Maximize fairness improvement per unit accuracy loss)

This approach is computationally cheap and interpretable: the tree structure (and thus its explanations) remain unchanged; only the terminal predictions are adjusted.

Exercise 3.2 — Same Weights

Do instances 1 and 10 get the same weight? Do instances 5 and 6?

Show Answer

Yes to both. Instances with the same (A, Y) combination always get the same weight. Instance 1 and 10 are both (Dutch, +); instances 5 and 6 are both (Non-Dutch, −). Weight depends only on the group-label cell, not on the individual instance.

Exercise 3.4 — Reject Option Classification

Using the dataset from Exercise 3.3 with $\theta = 0.6$. Which instances change from positive to negative?

Show Answer

Selection rates (original predictions, threshold 0.5): Dutch = 4/5 = 0.8 (after relabeling: instances 1,3,4,10 positive among Dutch non-relabeled), Non-Dutch = 1/4 = 0.25. Dutch is the favored group.

Exercise 3.5 — Leaf Relabeling

A tree has 20 instances (10 per group). Leaf $l_5$ has 2 negative instances (1 from each group). What happens if $l_5$ is relabeled to positive?

Show Answer

Δacc = −2/20 = −0.1 (2 true negatives are now misclassified).

Δdpd = 0. Each group gains exactly 1 positive prediction: +1/10 for both groups, so the difference in selection rates is unchanged.

Exercise 3.6 — Equalized Odds

Which techniques can optimize for equalized odds?

  • (a) Representation Learning
  • (b) Adversarial Learning
  • (c) Group-Specific Thresholds
  • (d) Relabeling
Show Answer

(b) and (c). Adversarial learning can optimize for equalized odds by giving the adversary both $\hat{Y}$ and $Y$. Hardt et al. explicitly optimize group-specific thresholds for equalized odds.
