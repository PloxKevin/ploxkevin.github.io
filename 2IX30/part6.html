LIME

LIME (Local Interpretable Model-agnostic Explanations) explains individual predictions of any black-box model by fitting a locally faithful interpretable model in the neighbourhood of the instance to be explained.

6-Step Procedure

  1. Select instance $x$ to be explained.
  2. Perturb $x$ to generate neighbourhood samples $z \in \mathcal{Z}$.
  3. Get black-box predictions $f(z)$ for all perturbed samples.
  4. Weight samples by proximity: $\pi_x(z) = \exp(-D(x,z)^2/\sigma^2)$
  5. Fit LASSO on the weighted samples to obtain a sparse linear model.
  6. Return the interpretable local model as the explanation.
Proximity Kernel

The kernel function used to assign weights to perturbed samples based on their distance from the original instance:

$$\pi_x(z) = \exp\!\left(-\frac{D(x,z)^2}{\sigma^2}\right)$$

Samples close to $x$ receive weight near 1; samples far away receive weight near 0. The bandwidth parameter $\sigma$ controls the width of the locality.

Fidelity is measured by local $R^2$ — how well the local linear model fits the black-box predictions in the neighbourhood. LIME uses perturbations of the input, which means generated samples may fall outside the training data distribution (the OOD problem). LASSO regularization ensures selectivity (sparse, few-feature explanations), but does not guarantee consistency across different runs or models.

For images, LIME uses superpixels as interpretable features — perturbation means masking superpixels on or off. This makes the explanation visual: highlighted regions show which parts of the image drove the prediction.

SHAP

SHAP (SHapley Additive exPlanations) attributes the prediction for a given instance to each individual feature, using a game-theoretic foundation from cooperative game theory.

Core Property

Every prediction decomposes into a baseline plus the sum of feature attributions:

$$f(x) = \mathbb{E}[f] + \sum_{i=1}^{k} \phi_i$$

Here $\mathbb{E}[f]$ is the mean prediction over the training data (the baseline), and each $\phi_i$ is the Shapley value of feature $i$ — its average marginal contribution across all possible feature coalitions.

Shapley Value Formula

The Shapley value of feature $i$ is defined as the weighted average of its marginal contributions over all subsets $S$ of the other features:

$$\phi_i = \sum_{S \subseteq N \setminus \{i\}} \frac{|S|!\,(n-|S|-1)!}{n!}\bigl[v(S \cup \{i\}) - v(S)\bigr]$$

$N$ is the full set of features, $n = |N|$, and $v(S)$ is the model's expected output when only the features in $S$ are observed (all others marginalized out).

Four Shapley Axioms

Property Meaning
Efficiency Shapley values sum to the difference between the actual prediction and the baseline: $\sum_i \phi_i = f(x) - \mathbb{E}[f]$. No contribution is lost or invented.
Symmetry Two features that make identical marginal contributions in every coalition receive the same Shapley value.
Dummy A feature that never changes the prediction in any coalition receives a Shapley value of zero.
Additivity For an ensemble of models, the Shapley value of the ensemble equals the sum of Shapley values from each constituent model.

Computational cost: Exact Shapley values require evaluating all $2^d$ feature subsets, which is exponential in the number of features $d$. TreeSHAP exploits the tree structure to compute exact Shapley values in polynomial time for tree-based models (random forests, gradient boosting, etc.).

Common Misconception

SHAP values are NOT regression coefficients. They are marginal contributions in a game-theoretic sense. The efficiency axiom provides a built-in correctness check: if $\mathbb{E}[f] + \sum_i \phi_i \neq f(x)$ for any instance, there is a bug in the implementation.

LIME vs SHAP

Both methods produce local, additive feature attributions, but differ fundamentally in their approach, theoretical grounding, and what question they answer. They can and do disagree on the same instance.

Dimension LIME SHAP
Approach Sensitivity — perturb the input and observe how predictions change Ablation — mask/marginalize features and measure their contribution
Question answered Which features matter most for a local linear approximation near $x$? What is each feature's average marginal contribution to the prediction at $x$?
Baseline Implicit (neighborhood center) Explicit: $\mathbb{E}[f]$ over the training distribution
Theory Heuristic — no uniqueness guarantees; results depend on kernel choice and sampling Axiomatic — Shapley values are the unique solution satisfying Efficiency, Symmetry, Dummy, and Additivity
Can they disagree? Yes — they answer different questions and may rank features differently on the same instance
Sign Disagreement Between LIME and SHAP

LIME and SHAP can disagree not just on magnitude but on sign — especially on non-linear functions. LIME measures local sensitivity (gradient-like), while SHAP measures average contribution (ablation-like). A feature that locally pushes the prediction up may have a negative average contribution overall.

Counterfactual Explanations

Definition

A counterfactual explanation is the smallest change(s) to input features that would change the prediction to a desired outcome.

Standard format: "If X had been v instead of v', the prediction would have been Y'."

Counterfactual explanations are considered human-friendly because they are contrastive — they explain why outcome A occurred rather than outcome B, which matches the way humans naturally seek and process causal explanations.

Eight Desirable Properties

Property Meaning
Validity The counterfactual must actually achieve the desired prediction outcome.
Proximity The counterfactual should be as close as possible to the original instance (few, small changes).
Actionability Suggested changes must be feasible for the individual to make (e.g., not "change your age").
Sparsity Explanations should change as few features as possible to remain interpretable.
Diversity Multiple distinct counterfactuals should be provided, offering different actionable paths.
Plausibility The counterfactual instance should lie in a realistic region of the input space (on-manifold).
Causality Changes should respect known causal structure between features (e.g., education before occupation).
Robustness Small perturbations to the counterfactual should not flip the explanation's conclusion.

Wachter Optimization

The Wachter et al. method finds a counterfactual $x'$ by minimizing a two-term loss that trades off prediction validity against proximity to the original instance:

$$\mathcal{L} = \lambda\bigl(f(x') - y'\bigr)^2 + d(x,\, x')$$

The regularization parameter $\lambda$ is increased iteratively until the counterfactual $x'$ achieves the desired target prediction $y'$. Limitation: this formulation does not incorporate actionability constraints — it may suggest changes to immutable features.

Instance-Based Counterfactual Explanations

An alternative to optimization: retrieve similar examples from the dataset that have different outcomes. Limitations: dependent on dataset composition, may lack diversity or plausibility.

Counterfactual Fairness Auditing

Counterfactual explanations can be used as a fairness audit tool. The procedure is:

  1. Generate $n$ counterfactual explanations per instance for a representative sample of the population.
  2. Compare the cost to flip (e.g., distance $d(x, x')$ or number of feature changes) across demographic groups.
  3. If one group systematically requires a larger cost to flip, the model imposes a disparate burden — a form of unfairness.

OOD Problem

Warning: Affects All Perturbation-Based Methods

The out-of-distribution (OOD) problem affects all perturbation-based explanation methods: LIME, SHAP, PDP, and ICE. When features are perturbed independently, the resulting synthetic instances may represent impossible or highly unrealistic feature combinations that never appear in the training data.

The model's behavior on OOD inputs is undefined — the model was not trained to make meaningful predictions there. Explanations derived from such predictions may be misleading, as they describe how the model behaves in regions of the input space it has never seen.

Example: Perturbing features independently can produce absurd combinations such as:

OOD Example

education = "preschool"  &  occupation = "database architect"

This combination is essentially impossible in reality. A model forced to predict on this input will interpolate or extrapolate in an uncontrolled way, and any explanation derived from it is not grounded in meaningful model behavior.

Mitigation strategies include generating perturbations that respect the empirical joint distribution of features (e.g., using conditional sampling or in-distribution perturbations), at the cost of increased computational complexity.

Exercises

Exercise 6.1 — SHAP efficiency check

Base value $\mathbb{E}[f]=0.8$, prediction $f(x)=0.64$. SHAP values: study_time=+0.03, resits=−0.03, attendance=−0.05, practice=+0.05. Is this correct?

Show answer
Check efficiency: $\mathbb{E}[f]+\sum\phi = 0.8 + (0.03-0.03-0.05+0.05) = 0.8+0 = 0.8$. But $f(x)=0.64 \neq 0.8$. There must be a bug — the efficiency property is violated.
Exercise 6.2 — Which techniques use perturbations?

Which techniques use perturbations? (a) SHAP  (b) LIME  (c) Logistic regression  (d) Global surrogate

Show answer
(a) and (b). SHAP uses perturbations for marginal contributions. LIME perturbs for neighborhood instances. Logistic regression is intrinsically interpretable. Global surrogates are trained on original data.
Exercise 6.3 — Best description of LIME’s intuition

Best description of LIME’s intuition? (a) LIME doesn’t suffer from OOD  (b) LIME has higher fidelity than global surrogates  (c) Black-box can be approximated locally with simpler model  (d) LIME captures interactions

Show answer
(c). LIME DOES suffer from OOD (rules out a). Fidelity claims are scope-dependent (b misleading). LIME typically doesn’t capture interactions (rules out d).
Exercise 6.4 — XAI method for a rejected loan

A bank rejects a customer’s loan. Suggest an XAI method.

Show answer
Counterfactual explanations — “If your income had been X, your loan would have been approved.” Actionable, contrastive, satisfies right to explanation. Alternatively, SHAP could show which features contributed negatively.
Exercise 6.5 — What do SHAP and PDP have in common?

What do SHAP and PDP have in common? (a) Both global  (b) Both intrinsic  (c) Both capture interactions  (d) Both suffer from OOD

Show answer
(d). Both rely on perturbations creating unrealistic feature combinations. SHAP is local (not a). Neither is intrinsic (not b). PDP doesn’t capture interactions (not c).
