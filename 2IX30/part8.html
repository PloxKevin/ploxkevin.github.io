k-Anonymity

Definition — k-Anonymity

Every quasi-identifier combination appears in at least k rows of the released dataset. Achieved via generalization (replacing specific values with ranges, e.g. age 34 → 30–39) and suppression (removing rare values entirely).

Limitations:

Limitation Description
Homogeneity attack If all k records in a group share the same sensitive attribute value, the attacker learns that value with certainty.
Background knowledge attack Prior information about the target can eliminate candidates within the k-group, leading to re-identification.
Utility loss Aggressive generalization and suppression degrade data quality, especially for small populations or rare attributes.

Differential Privacy

Definition — ε-Differential Privacy

A randomized mechanism $\mathcal{M}$ is $\epsilon$-differentially private if for all neighbouring datasets $d, d'$ (differing in exactly one row) and all measurable output sets $S$:

$$P(\mathcal{M}(d) \in S) \leq e^{\epsilon} \cdot P(\mathcal{M}(d') \in S)$$

Intuition: an adversary observing the output cannot reliably determine whether any single individual's data was included. This provides plausible deniability for every participant.

Privacy Budget (ε)

$\epsilon$ (epsilon) controls the trade-off between privacy and utility. Smaller $\epsilon$ means stronger privacy but more noise. In practice, values in the range 0.5–5 are most common.

ε value Privacy level Noise level Utility level
0 Perfect (infinite privacy) Infinite None (output is random)
Small (e.g. 0.1–0.5) Very strong High Low
Medium (e.g. 1–5) Moderate (practical range) Moderate Moderate to high
Large (e.g. >10) Weak Low High
∞ No privacy None Perfect (exact output)

Sensitivity

Definition — Global Sensitivity

The global (L1) sensitivity of a query $f$ measures the maximum change in output when one record is added or removed:

$$\Delta f = \max_{d,d'} \|f(d) - f(d')\|_1$$

Sensitivity determines how much noise must be added to achieve $\epsilon$-DP. Higher sensitivity requires more noise.

Query type Sensitivity ($\Delta f$) Explanation
Counting query 1 Adding or removing one person changes the count by at most 1.
Sum query (values in $[0, R]$) $R$ One person's contribution is at most $R$.
Mean query (values in $[0, R]$, $N$ people) $R/N$ Adding one person with value $R$ shifts the mean by at most $R/N$.

Laplace Mechanism

Definition — Laplace Mechanism

To release a query $f(d)$ with $\epsilon$-DP, add independent Laplace noise:

$$\mathcal{M}(d) = f(d) + \mathrm{Lap}(0,\, b), \quad b = \frac{\Delta f}{\epsilon}$$

The Laplace distribution $\mathrm{Lap}(0, b)$ has mean 0 and scale $b$. Larger $b$ means more noise. The mechanism satisfies $\epsilon$-DP because the likelihood ratio of any output under neighbouring datasets is bounded by $e^{\epsilon}$.

Laplace Distribution — PDF

The Laplace distribution has PDF:

$$p(x) = \frac{1}{2b}\exp\left(-\frac{|x|}{b}\right)$$

Key Properties

Property Statement Implication
Post-processing If $\mathcal{M}(d)$ is $\epsilon$-DP, then $g(\mathcal{M}(d))$ is also $\epsilon$-DP for any function $g$. Privacy cannot be weakened by downstream analysis. No need to protect outputs further.
Group privacy If $\mathcal{M}$ is $\epsilon$-DP, it is $k\epsilon$-DP for groups of $k$ individuals. Privacy guarantee weakens linearly with group size. Individuals in large groups have weaker protection.
Composition Running $k$ independent $\epsilon$-DP mechanisms costs $k\epsilon$ total privacy budget. Budget must be split across queries. Each of $k$ queries uses $\epsilon/k$, noise $b = k\cdot\Delta f/\epsilon$.
Key Insight

Dwork & Roth: "Too many accurate answers will reduce privacy spectacularly." Composition means every query consumes budget — splitting $\epsilon$ over $k$ queries multiplies noise by $k$. There is a fundamental tension between the number of queries answered and the accuracy of each answer.

Privacy Protection Hierarchy

De-identification (weakest) → defeated by Linkage attack
↓
k-Anonymity → defeated by Homogeneity attack, Background knowledge attack
↓
l-diversity / t-closeness → defeated by Reconstruction attack
↓
Differential Privacy (strongest formal guarantee)
  → uses Laplace Mechanism (noise ~ Δf/ε)
  → Post-processing property: cannot weaken after release
  → Composition property: budget consumed by each query
  → Group privacy: k rows → k·ε cost

Exercise 8.1 — k-Anonymity

Given quasi-identifiers: Gender, Age

NameGenderAgeCondition
AliceF22Flu
BobM24Diabetes
CarolF22Cancer
DaveM35Flu
EveF35Flu
FrankM24Diabetes

(a) Current k-anonymity? (b) Propose generalization for 3-anonymity. (c) Homogeneity attack risk after 2-anonymity?

Show answer
(a) Groups: (F,22)={Alice,Carol}, (M,24)={Bob,Frank}, (M,35)={Dave}, (F,35)={Eve}. Smallest group = 1. k = 1.
(b) Suppress gender: (F)={Alice,Carol,Eve}=3, (M)={Bob,Dave,Frank}=3. k = 3.
(c) (M,24) group: Bob and Frank both have Diabetes. Yes — homogeneity attack.
Exercise 8.2 — Differential Privacy calculation

500 employees, salaries [0, 200000], ε = 2. (a) Sensitivity? (b) Laplace noise parameter? (c) Also compute avg age [0, 70], same total ε = 2?

Show answer
(a) Δf = 200000 / 500 = 400.
(b) b = 400 / 2 = 200.
(c) Two queries, each ε/2 = 1. Salary: b = 400 / 1 = 400. Age: Δf = 70 / 500 = 0.14, b = 0.14 / 1 = 0.14.
Exercise 8.3 — Group privacy

An employee contributes 3 rows. How does this affect privacy?

Show answer
Group privacy: guarantee degrades to $e^{3\varepsilon}$. Effective cost for this individual is 3ε.
Exercise 8.4 — Post-processing property

You publish a DP average, then compute the median with another public statistic. Still ε-DP?

Show answer
Yes. Post-processing property: any function applied to ε-DP output remains ε-DP.

Practice Problems

Problem 3 — DP with multiple queries and group privacy

Average age of 1000 people, ages [0, 100], ε = 1.

(a) Sensitivity? (b) Noise parameter b? (c) Also compute avg income [0, 200000], total ε = 1? (d) Each person has 2 records?

Show answer
(a) Δf = 100 / 1000 = 0.1.
(b) b = 0.1 / 1 = 0.1.
(c) Two queries, each ε/2 = 0.5. Age: b = 0.1 / 0.5 = 0.2. Income: Δf = 200000 / 1000 = 200, b = 200 / 0.5 = 400.
(d) Group privacy: 2ε effective cost for that individual; sensitivity doubles as one person can shift the dataset by 2 records.
